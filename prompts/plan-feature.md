# Plan Feature

Deprecated compatibility prompt.

Use `prompts/grill-me.md` as the default planning and critique entrypoint before
non-trivial SoundAtlas feature, UX, data, documentation, route, enrichment, or
workflow work.

Use `soundatlas-implementation-planning` when the selected next step is to
create or update durable GitHub Issue planning content, including:

* Intake Issues
* Plan Updates
* Detailed Plan Updates
* Implementation Reports

Core rule:

> Do not implement directly from a vague prompt. Grill the request first, then
> capture non-trivial planned work in a GitHub Issue and implement only after
> explicit wording such as `implement issue #<number>`.

This file remains only as a backward-compatible alias for older references.
